* Phân tích trang web bán hàng :

- Front-end :

	+ Trang chủ.
	+ Trang danh sách sản phẩm.
	+ Trang giỏ hàng.
	+ Trang checkout.
	+ Trang thanh toán hoàn thành.
	+ Trang Liên hệ.

- Back-end :

	+ Tài khoản người dùng:

		Quản lý role(admin, user).
		Quản lý người dùng : Admin/user.
			Hiển thị/thêm/sửa/xóa.
		Đăng kí tài khoản.
		Đăng nhập tài khoản.

	+ Quản lý danh mục sản phẩm.
	+ Quản lý sản phẩm.
	+ Quản lý đơn hàng.

		Hiển thị danh sách đơn hàng -> Hiển thị giảm dần theo tgian.
			(Quản lý được trạng thái đơn hàng).

	+ Quản lý phản hồi.


* Phân tích thiết kế:

1. Bảng Role:

	- id: int -> Khóa chính tự tăng.
	- nameRole: String (varchar)

2. Bảng User (Phụ thuộc vào nghiệp vụ, chúng ta cần quản lý cái gì thì chúng ta sẽ đưa thông tin người dùng vào bảng. Không cần phải lấy quá nhiều thông tin vào)

	- id: int -> Khóa chính tự tăng
	- fullName: String
	- email: String
	- phoneNumber: String	(Nên sử dụng String cho phoneNumber).
	- address: String 
	- password: String -> Sử dụng md5.
	- roleID: int -> foreign key. 	//Mỗi người dùng sẽ có 1 role.
	- created_at: datetime -> tạo bản ghi.
	- updated_at: datetime -> sửa bản ghi.

3. Bảng Category (Bảng danh mục, phân loại sản phẩm thành các mục).

	- id: int -> Khóa chính tự tăng.
	- nameCategory: String

4. Bảng Product (Bảng sản phẩm).

	- id: int -> Khóa chính tự tăng.
	- categoryID: int -> foreign key. 	//Mỗi sản phẩm sẽ thuộc 1 loại danh mục.
	- nameProduct: String.
	- price : int
	- discount : int
	- thumbnail : String.
	- description: longtext.
	- created_at: datetime -> tạo bản ghi.
	- updated_at: datetime -> sửa bản ghi.

5. Bảng quản lý galery (Bởi vì 1 sản phẩm sẽ có nhiều hình ảnh, nên chúng ta cần phải quản lý các hình ảnh con đó).

	- id: int -> Khóa chính tự tăng.
	- productID: int -> foreign key.
	- thumbnail: String

6. Bảng quản lý phản hồi _ feedback

	- id: int -> Khóa chính tự tăng.
	- firstName: String
	- lastName: String
	- email: String
	- phoneNumber: String
	- subjectName: String
	- note: String

7. Bảng quản lý đơn hàng.

	- id: int -> Khóa chính tự tăng.
	- fullName: String
	- email: String
	- phoneNumber:
	- address:
	- note:
	- orderDate: datetime -> Thời điểm đặt hàng.
	- Danh sách sản phẩm: 
		- sản phẩm 1 * số lượng * giá tại thời điểm mua.
		- sản phẩm 2 * số lượng * giá tại thời điểm mua

	7.1 Bảng order:

		- id: int -> Khóa chính tự tăng.
		- fullName:
		- email:
		- phoneNumber:
		- address:
		- note:
		- orderDate: datetime -> Thời điểm đặt đơn hàng.
		- status: int -> (pending, approved,....)
		- totalMoney: int -> Tổng tiền của đơn hàng.



	7.2 Bảng chi tiết đơn hàng:

		- id: int -> Khóa chính tự tăng.
		- orderID: int -> foreign key -> Order(id)
		- productID: int -> foreign key -> Product(id).
		- price: int
		- num: int
		- totalMoney: Lưu tổng tiền của sản phẩm (price*num).
